General Information

General Information section explains in general terms the system and the purpose for which it is intended.

System Overview

Mobile trust is application, which allows storing web URL, and their passwords, Generate strong passwords and Generate One-time password. Mobile trust users can login to the application, add, edit, delete and search Password Vault.

Mobile trust operates on mobile devices with IOS and Android operating system.

System Summary

System Summary section provides a general overview of the system. The summary outlines the uses of the system’s hardware and software requirements, system’s configuration, user access levels and system’s behavior in case of any contingencies.

System Configuration

Mobile trust operates on mobile devices with Android and IOS operating system. It is compatible with Android 2.2 (and higher versions) and IOS 5.1(and higher versions). The application requires connection to Internet in order to enable feature like secure browsing, launch web page from password vault and one time password generation but mostly stores data locally. After installation on the device, Mobile trust can be used immediately without any further configuration.

User Access Levels

Only registered users can use application and are able to save data to database.

Strong password generator

Description
Stronger password generator is password suggestion utility for Mobile trust user. User can generator strong password.

User can click on strong password generator in Main Menu screen (as shown in above screen). This action will lead user to below screen

Password Length

User can mention length of desired password (which he wants to generate)

Lower Case on/off

User can set Lower case on/off. If user sets lower case on than there may be lower case characters in application generated password and vice versa if user sets lower case off than there won’t be any lowercase character in application generated password.

Upper case on /off

User can set Upper case on/off. If user sets upper case on than there may be upper case characters in application generated password and vice versa if user sets upper case off than there won’t be any uppercase character in application generated password.

Numbers

User can set numbers on/off. If user sets numbers on than there may be numbers in application generated password and vice versa if user sets numbers off than there won’t be any numbers in application generated password.

Non Alpha-Numeric

User can set Non-Alpha-Numeric on/off. If user sets Non Alpha-numeric on than there may be Non Alpha-numeric characters (Special Characters) in application generated password and vice versa if user sets non Alpha-numeric off than there won’t be any Non Alpha-numeric characters (Special characters) in application generated password.

One time password Generator

Description - one time password generator is based on ‘open ID’. Open ID is a widely used open protocol standard that enables users to authenticate themselves in a decentralized manner and does not rely on a central authority to authenticate a user's identity. Open ID enables users to consolidate their digital identities. Users may create accounts with their preferred OpenID identity providers, and then use those accounts as the basis for signing on to any website which accepts OpenID authentication.

User need to click on one-Time password generator on main menu (screen shown as above)

Above user action will lead user to first screen (shown below). User needs to click on New Entry (in first screen) and that will lead user to second screen (shown below)

Now user can generate one-time password using two methods

Add New ProtectID OTP

Click on Add NEW ProtectID OTP (Shown in above screen) will lead user to below screen

User can register himself through above screen

User need to fill up fields mentioned in above screen (fields with * mark are mandatory fields)

Field PIN is numerical field and user need to put any 4-8 digit numbers in this field.

Field Method is used by user to choose algorithm to generator one time password.

Register

User can click on register button to initiate registration process. During registration process Application will perform following actions

Web method is called with Username, Password, Company, PIN (Web Pin), Phone, Ticket, Locale and Device ID fields. On successful execution of web method, it returns an URL corresponding to a specific customer and if error occurred it returns an error message. Using the URL obtained another web method is called.

The web service first validates the data for unique UserID and DeviceID, provisions (registers) the data in Server and returns back Device PIN. If an error occurred or registration process is not successful for any reasons it returns an Error message.

Clicking on register button will lead user to below screen

User need to provide Profile name and then click on next button in above screen. This action will lead user to below screen

Now user need to click on Get OTP and this will lead user to next screen (shown as below)

User can perform two actions on above mentioned second screen

Click on Get OTP

Get OTP will generate new One-Time password locally using one of the method called

  • Event based algorithm
  • Time based algorithm

Algorithm will be considered based on DevicePIN returned from the server. Whenever DevicePIN will be returned blank then Time based algorithm used otherwise Event based algorithm used.

Get OTP will use DeviceID & UserID for secret key to generate new OTP locally and that OTP will use to authenticate on the server. Back button will redirect the user to profile list screen.

Click on Resync

The device may go OUT OF SYNC with the register data in the System, because of several unsuccessful authentication tries or repeated generation of One-Time Password by the end user (without using the generated values for authentication). In this case the user can perform a ‘ReSync’ operation to synchronize the application with the Server web Service.

Settings

User need to navigate as mentioned in below screens to change settings.

Secure Key Board

User can set secure key board on/off . If user sets Secure key board on Application will provide custom made key board instead of device’s standard key board and vice versa if user sets securekey board off application will provide user device’s standard key board

Save Password

User can make 'save password' on/off. While 'save Password' on, App will automatically perform login activity for user(no need to type user name/password). On "save Password" off, user need to put user name ans password and click on login button manually to login

Backup my data

User can take Application data back up through data Backup button. Encrypted Data with password protection will be moved to following location

  • In Android device - android: /sdcard/mobiletrust/archive.zip
  • In IOS device - IOS: itunes -> device -> Apps -> File Sharing/archieve.zip
Restore My Data

User can restore earlier back up through restore button
